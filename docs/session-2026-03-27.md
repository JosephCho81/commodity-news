# 세션 1 — 멀티 클라이언트 플랫폼 설계 + Perplexity 검증 (2026-03-27)

## 목표
코리아에이원 원자재 뉴스 서비스를 멀티 업체 플랫폼으로 고도화.
- path 기반 업체별 개인화 (`/:clientId`)
- 코리아에이원 브랜딩 유지, 내용만 업체별 차별화
- 신규 원자재: 철강, 유연탄(원료탄), 열탄, 고철/스크랩

## 확정된 설계 원칙

### 아키텍처
- 클라이언트 설정: `clients.config.ts` (코드 기반, 관리 UI 없음)
- 라우팅: `/:clientId` → `clients.config.ts` 기반 동적 렌더링
- Firestore 네임스페이스: `/clients/{clientId}/data/{tab}`
- 인증: 없음 (URL = 접근)

### 마이그레이션 원칙
- Phase 1: 신규 경로만 추가, 기존 코드 무수정
- Phase 2: 코리아에이원도 clients.config.ts로 통합
- Phase 3: 기존 코드 정리

### 신뢰성 원칙
- 확인된 데이터만 표시. null = UI 미노출
- 추정값·범위 추정 절대 금지

## Perplexity 검증 결과

테스트: sonar, temperature 0.1, max_tokens 3000, "찾으면 보고, 못 찾으면 null, 추정 금지"

| 원자재 | 추출 성공 | 실패 이유 |
|--------|----------|----------|
| 철강 HRC | 중국 FOB 472, 미국 835, EU 665 | 터키 내수 → 구독형 |
| 원료탄 | PLV HCC 223.5 (SGX 선물) | CFR → 구독형 |
| 열탄 | Newcastle 텍스트 언급만 | 벤치마크 필드 전부 null |
| 고철 | HMS 터키 380, 일본 49,000 JPY | US Shredded → 구독형 |

**핵심 발견**: 공개 데이터 추출 가능, 구독형 지수 접근 불가.
**미해결**: "빈 값 제공 불가" vs "확인된 데이터만 표시" 충돌.
